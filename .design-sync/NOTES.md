# design-sync notes — vidgen

Repo-specific gotchas for future syncs of this design system.

## Setup facts

- **This is an app, not a published library.** No `dist/` component entry, no
  `node_modules/frontend`. The converter runs in **synth-from-src** mode, so the
  build needs `--entry frontend/src/ui/index.ts` (the barrel) to locate the
  package root — omitting `--entry` fails with `node_modules/frontend/package.json ENOENT`.
- **`srcDir: src/ui`** scopes discovery to the pure primitives. This deliberately
  excludes `main.tsx` (its top-level `createRoot`/`connect()` would throw at
  bundle-eval) and every store-coupled component in `src/components/**` (they
  read the Zustand store and can't render standalone).
- Components are pinned explicitly in `componentSrcMap` — synth-mode auto-discovery
  found 0 (no `.d.ts` tree), so each primitive is listed.
- `.d.ts` prop extraction yields `[key: string]: unknown` in synth mode, so real
  props are hand-written in `dtsPropsFor`. **Keep `dtsPropsFor` in sync when a
  primitive's props change** — it will not auto-update.

## CSS pipeline (important)

- `css.mjs` **copies `cssEntry` verbatim** and does NOT resolve `@import`s, and
  `tokensGlob` only works with a `tokensPkg`. So the repo's modular
  `src/styles/{tokens,base,components}.css` must be pre-bundled into ONE
  self-contained file. That is what `buildCmd` does (esbuild `--bundle` →
  `frontend/dist/ds-styles.css`), and `cssEntry` points at that output.
- **Re-run `buildCmd` (or the driver, which does) whenever any `src/styles/*.css`
  changes**, before `package-build.mjs` — otherwise the bundle ships stale CSS.
- `fonts.css` is kept OUT of `index.css` on purpose (so the esbuild cssEntry
  bundle stays asset-free). It ships via `cfg.extraFonts`; the app imports it
  directly in `main.tsx`.

## Provider

- `cfg.provider = Surface`. The DS is dark-themed; preview cards force a white
  background, so without the Surface wrapper components render light-on-white.
  Surface is also the real app theme root.

## Render verification

- Playwright/Chromium was **not** installed. Renders were verified with
  **agent-browser** (accessibility snapshot + `getComputedStyle` eval) against a
  local `http-serve` of `ds-bundle/` — confirmed all 9 cards mount (no floor-card
  fallback), amber accent (`oklch(0.8 0.14 82)`), dark surface, Hanken Grotesk.
  `package-validate.mjs` was run with `--no-render-check`.
- `agent-browser screenshot` HANGS in this environment; `snapshot` and `eval`
  work. Use those for verification, or install Playwright for the real gate.

## Known render warns

- `[RENDER_SKIPPED]` — expected while `--no-render-check` is used (no Playwright).

## Re-sync risks

- **Font**: Hanken Grotesk latin woff2 was fetched from jsdelivr fontsource at
  sync time and committed under `frontend/src/styles/fonts/`. It's in-repo now,
  so no network dependency on re-sync.
- **dtsPropsFor drift**: hand-written props can silently diverge from the real
  component props — re-check against `src/ui/*.tsx` on any primitive API change.
- **cssEntry staleness**: `dist/ds-styles.css` is a generated artifact (gitignored
  via `dist/`); it must be regenerated by `buildCmd` before every build.
