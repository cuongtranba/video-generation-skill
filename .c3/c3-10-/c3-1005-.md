## Goal

Consume VIDGEN_EVENTS via a durable NATS consumer and fold every event into Postgres tables for the HTTP read path.

## Purpose

Owns applyProjection (switch over all 12 VidgenEvent types), runProjections (long-running durable consumer loop), and rebuildProjections (wipe + replay from seq 0). Tables: projects, scenes, assets, cost_ledger. applyProjection stays a pure read-model fold; runProjections additionally accepts an optional live-consume reaction (api/src/reactions.ts) run AFTER each fold — used to dispatch the caption job once every scene's VoiceSynthesized has landed. The reaction runs only on the live path, never in rebuildProjections, so a read-model rebuild never re-dispatches jobs. Non-goal: does not own the source of truth — VIDGEN_EVENTS is; Postgres is a disposable, rebuildable read model.
